- **File Descriptor와, File System에 대해 설명해 주세요.**
  - **File Descriptor**는 운영체제에서 파일과 같은 자원을 추상적으로 표현하는 정수 값입니다.
    - 리눅스 혹은 유닉스 계열의 시스템에서 **프로세스가 파일을 다룰 때 사용하는 개념**.
    - 프로세스에서 **특정 파일에 접근할 때** 사용하는 추상적인 값. (일반적으로 0이 아닌 정수값)
    - 프로세스가 파일을 열면 (OS의 open() 시스템 호출) ⇒ 파일 디스크립터 반환
      - read(), write(), close()도 마찬가지
  - **File System**은 운영체제가 **파일을 저장, 관리, 접근할 수 있도록 하는 구조와 매커니즘**이라고 할 수 있습니다.
    - 다시 말해, SSD나 HDD와 같은 데이터를 저장하는 디스크 위에서 파일을 구성하는 방식을 정의하는 것입니다.
      - EXT4 (Linux), NTFS (Windows), exFAT (이동식 저장장치)
    - 파일 시스템의 주요 기능은 다음과 같습니다.
      - 파일 저장 및 관리 : 데이터를 파일 단위로 저장하고, 디렉터리를 통해 파일을 조직화
      - 이름과 메타데이터 제공 : 파일 이름, 크기, 소유권, 권한 등
      - 데이터 보호 : 접근 제어 (권한 설정), 암호화 등
      - 효율적인 공간 관리 : 블록 단위로 데이터 저장 → 공간 최적화 → 디스크 사용률 높임
-----

- **I-Node가 무엇인가요?**
  - I-Node는 파일의 메타데이터를 저장하는 데이터 구조체입니다.
    - 파일의 실제 데이터는 블록 단위로 저장됩니다. 이때, I-Node는 해당 파일이 어디있는지를 관리합니다.
    - 파일 시스템에서 파일 이름을 통해 I-Node를 찾아 파일 데이터에 접근합니다.
  - I-Node는 파일 자체의 정보를 관리하지만, 파일 이름은 디렉터리에서 따로 관리됩니다.
  - 같은 I-Node를 여러개의 파일 이름이 가리킬 수 있습니다. (하드 링크)
    - 일반적으로는 하나의 파일 당 하나의 I-Node가 할당. (심볼릭 링크는 예외적으로 새로운 I-Node를 가질 수 있음)
  - I-Node 기반의 파일 접근 과정
    - 파일 시스템이 디렉터리에서 파일 이름을 검색 → I-Node 번호를 찾음
    - 해당 I-Node를 읽어 데이터 블록의 위치 확인
    - 실제 데이터 블록을 읽어 파일 내용을 가져옴
